# 🧠 Chapter 2 — How Computers Think

Before learning syntax and writing programs, we need to understand **how a computer actually executes instructions**.

Programming becomes much easier when you understand **how computers think**.

---

# 🏗 The Computer is Like a Factory

Imagine a factory that processes instructions.

The factory has three main parts:


Input → Processing → Output


Every computer program works the same way.

Example:

Input → numbers from user  
Processing → add the numbers  
Output → show the result

---

# ⚙️ The Three Core Components of a Computer

Inside a computer there are three important components:

### 1️⃣ CPU (Central Processing Unit)

The CPU is the **brain of the computer**.

It performs calculations and executes instructions.

You can think of it as the **worker that follows your program instructions**.

---

### 2️⃣ Memory (RAM)

Memory stores data while the program is running.

For example:


int age = 20;


The value `20` is stored in memory.

The CPU reads from memory when executing the program.

---

### 3️⃣ Storage

Storage keeps data permanently.

Examples:

- Hard disk
- SSD

Programs are stored here before they run.

---

# 🔄 How a Program Runs

When you run a program, the computer follows these steps:


Load program from storage

Place program in memory

CPU reads instructions

CPU executes instructions

Output is generated


Every program you write follows this cycle.

---

# 📦 Example: Simple Program Execution

Consider a program that adds two numbers.


Ask user for number

Ask user for another number

Add numbers

Print result


The computer executes these instructions **one by one**.

It never skips steps.

---

# 🤖 Important Rule

Computers are **very fast but very literal**.

They do **exactly what you tell them**.

If a program has an error, the computer will not fix it.

It will simply follow the wrong instructions.

That is why programmers must write instructions **very carefully**.
